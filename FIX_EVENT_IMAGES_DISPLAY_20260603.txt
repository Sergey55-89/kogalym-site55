Исправление отображения фото афиши от 03.06.2026

Причина:
- Фото карточек выводились через CSS background-image. На GitHub Pages часть карточек могла отображаться тёмным блоком, если картинка не успевала загрузиться или путь не был обработан браузером как ожидается.

Что исправлено:
1. Карточки афиши теперь выводят фото через обычный тег <img>, а не только через background-image.
2. Для каждого фото добавлен резервный путь через data-fallback.
3. Если фото события не загрузится, карточка автоматически подставит резервную картинку по категории.
4. Для спектаклей оставлены локальные фото постановок в images/events.
5. Проверено: все картинки из events-data.json существуют внутри проекта.
6. Проверено: синтаксис скрипта страницы events.html валидный.
7. Проверено: синтаксис scripts/update-events.mjs валидный.
